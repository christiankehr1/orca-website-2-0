# Framer embeds — self-contained pages

Each `.html` file here is **one complete, self-contained page** — CSS, icon sprite,
nav/footer, fonts (base64), logos (base64) and all scripts are inlined. No hosting,
no external files. Paste one into a Framer **Embed** element and it renders identically
to the static site.

Generated by `../build-framer-embeds.js` — re-run `node build-framer-embeds.js` from the
project root any time you change anything in `site/`.

## How to place one in Framer

1. In Framer, create (or open) the page — e.g. a page at `/platform`.
2. **Insert → Embed** (or press the Insert menu and pick "Embed").
3. Set the embed type to **HTML** and paste the entire contents of the matching file
   (e.g. `platform.html`).
4. Size it: set the Embed to **100% width**. Give it a **fixed height** equal to the page's
   rendered height so nothing clips (the embed is a sandboxed iframe and won't auto-grow).
   Measured desktop heights (at 1200px wide):

   | Page | Height (px) |
   |---|---|
   | `index.html` | 6514 |
   | `platform.html` | 4087 |
   | `solutions.html` | 2811 |
   | `why-orca.html` | 2695 |
   | `segment.html` | 2578 |
   | `book-demo.html` | 2233 ¹ |
   | `watch-demo.html` | 1942 |
   | `customers.html` | 1764 |
   | `404.html` | 845 |

   ¹ `book-demo` grows once the SavvyCal widget finishes loading — give it ~250px of slack
   (or set the embed to clip-free / scroll). Heights are width-stable above ~1180px; below
   that the layout reflows (taller) — set per-breakpoint heights in Framer if you support mobile.

   Tip: it's easiest to make the embed the **only element** on the Framer page at 100% width,
   with the height above.

Repeat per page. File → Framer page mapping:

## Notes / caveats (inherent to the embed approach)

- **Sandboxed:** each embed is an iframe, so it won't share Framer's SEO/scroll.
- **Link routing is built in.** At runtime, cross-page links are rewritten to Framer page
  paths and given `target="_top"`, so clicking nav/footer/CTA links navigates the *parent*
  Framer site (not the iframe). Same-page `#anchors` (sub-nav, ladder) stay internal and
  scroll within the embed. **Assumed Framer slugs:** `/`, `/platform`, `/solutions`,
  `/segment`, `/why-orca`, `/customers`, `/watch-demo`, `/book-demo`. If your Framer page
  paths differ, edit `MAP` in `build-framer-embeds.js` (or tell me the slugs) and re-run the build.
- **Fonts** are embedded (Inter + Inter Display) so headings are pixel-exact with zero setup.
  That's what makes each file ~1.3 MB; it's a one-time paste.
- **SavvyCal** (on `book-demo.html`) loads live from savvycal.com — works as-is.

## Wiring the two live demos (optional, one step)

The Sonar walkthrough (`index.html`) and the structure chart (`platform.html`, `segment.html`)
are shown as **placeholders** in these files, because the demos are multi-file / 3.4 MB and
can't be inlined. To make them live:

1. Host the `site/demos/` folder anywhere static (your domain, Netlify, Vercel, GitHub Pages).
   Call that base URL `HOST` (e.g. `https://assets.withorca.com`).
2. In the embed HTML, find the placeholder block (`<div class="ph"> … Interactive Sonar
   walkthrough … </div>` or `… Live structure chart …`) and replace it with:

   **Sonar (index.html):**
   ```html
   <div class="embed" data-base="1440" data-ratio="0.66">
     <iframe src="HOST/demos/sonar-demo/interactive-sonar.html" title="ORCA Sonar" loading="lazy" scrolling="no"></iframe>
   </div>
   ```

   **Structure chart (platform.html, segment.html):**
   ```html
   <div class="embed" data-base="1456" data-ratio="0.62">
     <iframe src="HOST/demos/structure-chart.html" title="ORCA structure chart" loading="lazy" scrolling="no"></iframe>
   </div>
   ```

   The inlined `chrome.js` already scales these to fit (its `wireEmbeds()` runs automatically).
   Tell me your `HOST` and I'll bake the live demos straight into the generated files.
